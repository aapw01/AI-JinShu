# AI-JinShu 超长连载技术方案（讨论稿）

本文档用于支持目标：**几百章一次性全自动高一致性成稿**。  
当前阶段仅为方案，不涉及代码实施。

## 1. 目标与范围

| 项目 | 定义 |
|---|---|
| 目标 | 300+ 章节可持续自动生成，前后事实一致，风格稳定，可自动恢复失败任务 |
| 非目标 | 不以 token 成本优化为第一优先级；先保证效果 |
| 实施原则 | 保留现有 FastAPI/Celery/LangGraph 外壳，重构生成与记忆内核 |

## 2. 当前架构与差距

| 维度 | 当前能力 | 主要差距 |
|---|---|---|
| 编排 | LangGraph 章节循环可运行 | 整本单任务，缺少分卷调度与 checkpoint 治理 |
| 记忆 | 摘要窗口 + thread ledger + 向量检索 | 缺少强结构化事实层与版本管理 |
| 一致性 | 规则检查与重试 | 仍偏软约束，自动修复链路不完整 |
| 质量 | 基础审校 + 语言评分 | “好看”能力缺少可量化闭环 |

## 3. 全量改造项（确认清单）

| 模块 | 要做的点 | 目标 | 优先级 |
|---|---|---|---|
| 生成架构 | 全书单任务改为“分卷任务 + 总控任务” | 支撑几百章稳定执行 | P0 |
| 工作流 | 三层状态机（book/volume/chapter）+ checkpoint 恢复 | 可中断、可回放、可续跑 | P0 |
| 分层规划 | 全书弧线 -> 卷级计划 -> 章级计划，卷末强制 replan | 降低长程漂移 | P0 |
| 事实记忆 | 新建 Story Bible（角色/关系/地点/道具/规则/事件/伏笔）结构化存储 | 一致性“有真值源” | P0 |
| 版本管理 | 事实记录版本化（from/to chapter + revision） | 防止历史污染 | P0 |
| 时间线 | 统一世界时间线与事件线，自动冲突检测 | 消除时序穿帮 | P0 |
| 检索策略 | 结构化检索优先 + 向量补充 + 重排 | 提高上下文命中率 | P0 |
| 上下文装配 | 硬约束区/软参考区分离，硬约束不可改写 | 降低模型随意发挥 | P0 |
| 写后回写 | 每章后做事实抽取并回写 Bible | 让记忆持续可用 | P0 |
| 一致性门禁 | 章前 Gate + 章后 Gate，失败不入库 | 从提醒升级为阻断 | P0 |
| 自动修复 | Gate 失败触发：局部修复 -> 章重写 -> 卷重规划 | 提高全自动成功率 | P0 |
| 伏笔系统 | 伏笔 ID 生命周期（planted/resolved/expired） | 控制埋回节奏 | P1 |
| 文风一致性 | Style Bible + 角色声纹 + 叙事语气约束 | 减少后期“失味” | P1 |
| 爽点节奏 | 章节 beat sheet（钩子/冲突/兑现/反转）强约束 | 提升可读性与追更性 | P0 |
| 情绪张力 | 情绪曲线目标与偏差评分 | 稳定情绪推进 | P1 |
| 去重反套路 | 桥段/句式/套路重复检测，超阈值改写 | 提升新鲜感 | P1 |
| 双评审体系 | 事实评审器 与 审美评审器分离 | 兼顾“对”与“好看” | P0 |
| 质量指标 | 章级/卷级/全书级指标体系（一致性+吸引力） | 可量化迭代 | P0 |
| 评测集 | 长篇回归集（跨100/200/300章）+ 基准样本 | 防止回归 | P0 |
| A/B机制 | 同章多策略对比，按指标自动选优 | 持续提升成稿质量 | P1 |
| 人工反馈闭环 | 编辑偏好标注 -> 策略更新 | 拉高审美上限 | P1 |
| 数据库迁移 | 新表 + 索引 + 兼容旧数据读写 | 平滑升级 | P0 |
| 可观测性 | 记录每章决策、Gate 结果、修复路径、证据链 | 问题可定位 | P0 |
| 前端能力 | 展示卷级进度、Gate 报告、修复历史、质量评分 | 人机协作可控 | P1 |
| 上线策略 | 新旧 pipeline 并行灰度 + 回滚预案 | 降低上线风险 | P0 |
| 合规安全 | 内容安全、版权风险、敏感题材审查链 | 商用可持续 | P1 |

## 4. 推荐目标架构（保留外壳，重构内核）

| 层 | 新职责 |
|---|---|
| API 层 | 提供 book/volume/chapter 任务控制、状态查询、报告查询 |
| Orchestrator 层 | 总控图 + 分卷子图，统一 checkpoint 与恢复 |
| Planning 层 | 全书/卷/章三级规划器，卷末自动重规划 |
| Memory 层 | Story Bible（结构化事实）+ Vector Memory（语义补充） |
| Gate 层 | 章前约束检查、章后事实校验、卷末一致性验收 |
| Quality 层 | 事实评审 + 审美评审 + 节奏评审 |

## 5. 数据模型改造（新增核心表）

| 表 | 作用 | 关键字段 |
|---|---|---|
| `story_entities` | 角色/地点/组织/道具主实体 | `id`, `novel_id`, `entity_type`, `name`, `status`, `revision` |
| `story_facts` | 结构化事实条目 | `entity_id`, `fact_type`, `value_json`, `chapter_from`, `chapter_to` |
| `story_events` | 事件与因果链 | `event_id`, `chapter_num`, `actors`, `causes`, `effects` |
| `story_foreshadows` | 伏笔生命周期 | `foreshadow_id`, `planted_chapter`, `resolved_chapter`, `state` |
| `story_snapshots` | 卷级事实快照 | `volume_no`, `snapshot_json`, `created_at` |
| `generation_checkpoints` | 可恢复执行点 | `task_id`, `volume_no`, `chapter_num`, `node`, `state_json` |
| `quality_reports` | 章/卷/书质量评估结果 | `scope`, `scope_id`, `metrics_json`, `verdict` |

## 6. 关键流程改造

| 流程阶段 | 必做动作 | 产出 |
|---|---|---|
| 卷前 | 读取上卷快照，生成本卷 plan | `volume_plan` |
| 章前 | 组装硬约束与软参考，执行 pre-gate | `chapter_context` |
| 章生成 | writer -> reviewer -> revise loop | `draft/final_text` |
| 章后 | 抽取事实并回写，执行 post-gate | `fact_delta`, `chapter_report` |
| 卷末 | 汇总一致性与审美指标，不达标触发 replan | `volume_report`, `next_volume_plan` |

## 7. 质量上限专项（“好看”能力）

| 能力 | 工程化方案 | 门禁指标示例 |
|---|---|---|
| 爽点节奏 | 先产 `beat_sheet` 再写正文，正文对齐 beat | 钩子密度、兑现率、反转强度 |
| 情绪张力 | 章节情绪目标曲线 + 偏差评估 | 情绪偏差分 < 阈值 |
| 人物魅力 | 角色声纹约束 + 动机一致性检查 | 人设偏离分 < 阈值 |
| 新鲜度 | 去重/反套路检测 | 重复桥段率 < 阈值 |
| 审美质量 | 独立审美评审器 | 审美分 >= 阈值 |

## 8. 过拟合风险与防护

| 风险 | 防护措施 |
|---|---|
| 对固定评测集过拟合 | 每周刷新失败样本，动态扩充评测集 |
| 对固定模板过拟合 | 保留受控探索率，卷内策略多样化 |
| 对单模型过拟合 | 多模型交叉回归与灰度 |
| 指标刷分 | 联合指标门槛，不允许单指标优化 |

## 9. 实施分期（建议）

| 阶段 | 范围 | 结果 |
|---|---|---|
| Phase 1 | P0 中“事实层 + 分卷编排 + Gate 基线 + checkpoint” | 系统可稳定跑超长任务 |
| Phase 2 | P0 剩余“质量评估体系 + 回归评测集 + 可观测性” | 一致性显著提升、可迭代优化 |
| Phase 3 | P1 “审美能力与人类反馈闭环” | 成稿吸引力持续提升 |

## 10. 迁移与风险控制

| 项目 | 方案 |
|---|---|
| 迁移方式 | 直接切换到新 pipeline，不保留旧链路 |
| 回滚策略 | 回滚到上一版代码与数据库备份（不做运行时双轨回退） |
| 数据兼容 | 不做旧表兼容读写；按新模型重建/迁移 |
| 验收标准 | 100+ 章与 300+ 章回归套件通过后上线 |

## 11. 确认后执行入口

确认后按以下顺序启动代码改造：
1. `Phase 1` 的数据模型与迁移脚本。
2. 分卷调度与 checkpoint 子图。
3. 章前/章后 Gate 与事实回写链路。
4. 质量评估与回归自动化。
